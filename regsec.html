Open regedit.exe (press Win+R, type regedit, press Enter). Go to File → Export. Under Export Range, select All. Choose a save location, give it a name, and click Save. The resulting .reg file can then be uploaded to the REGSEC Registry Audit tool.
When you download software, the publisher typically provides a known-good SHA-256 or MD5 hash. REGSEC computes the real cryptographic hash of your downloaded file using the browser's Web Crypto API (for SHA-256/SHA-1) and a pure-JS MD5 implementation. If your file's hash doesn't match the publisher's, the file may have been modified, corrupted in transit, or replaced with malware.
The current browser-based version uses a deterministic simulation to demonstrate the interface. In a production deployment with API keys, REGSEC connects to real threat intelligence feeds from engines including ClamAV, Kaspersky, Bitdefender, CrowdStrike, SentinelOne, and more. Hash verification is fully real — those SHA-256, SHA-1, and MD5 values are genuine cryptographic hashes of your file.

Each check is weighted by severity.
